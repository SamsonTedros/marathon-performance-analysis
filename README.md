# Marathon Performance Analysis

## Overview
This project looks at my marathon data to understand how I paced the race, where I started to slow down, and whether that slowdown was caused by fatigue or the course itself.

## Tools Used
Python using Pandas and Matplotlib  
Google Colab  

## Dataset
The data is broken down by mile and includes pace, grade adjusted pace and elevation change.

## Key Findings
I averaged a 9.82 min per mile pace in the first half of the race but slowed to 11.97 in the second half which is about a 2.15 minute drop per mile  
My pace started to fall off around mile 17 or 18 and really broke down after mile 20  
When I compared my actual pace to grade adjusted pace both dropped in the later miles which shows the slowdown came from fatigue and not hills  
Overall the data shows I went out at a pace that I could not hold for the full race which led to a big drop in performance at the end  

## Files
marathon_performance_analysis.ipynb contains the full analysis and charts  
Marathon.csv is the raw data  

## Summary
This project shows how pacing affects endurance performance and how data can be used to break down what actually happened during a race
